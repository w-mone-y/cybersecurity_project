Web 安全

掌握 OWASP Top 10 与现代 Web 的防护策略

课程目录

1. OWASP Top 10 概览

聚焦于最关键的 Web 应用安全风险：访问控制失效、加密失败、注入、设计缺陷、认证失败、敏感数据暴露、软件和数据完整性失败、日志与监控不足等。

2. XSS 原理与防护

核心在于输出编码与上下文敏感：HTML、属性、URL、JS 不同上下文需不同编码策略；同时配合 CSP 限制内联脚本与域。

三类路径不同，但本质都是未受控输入进入可执行上下文。

  • 模板引擎自动转义
  • 严格 CSP（script-src、object-src）
  • 避免危险的 innerHTML，使用 textContent 或安全模板
  • 对 URL、属性、JS 上下文分别编码

3. CSRF 与 SameSite

CSRF 防护：SameSite=Lax/Strict、双提交 Cookie、CSRF Token、Referer/Origin 校验、对“有副作用”的请求使用 POST + Token。

4. 安全头部与 CSP

  • Content-Security-Policy
  • Strict-Transport-Security
  • X-Frame-Options / frame-ancestors
  • Referrer-Policy
  • Permissions-Policy
